# 全新实现编码规则

日期：2026-06-16

## 语言规则

- 所有 `.md` 文档使用中文书写。
- 所有代码注释使用中文书写。
- 函数名、类型名、宏名可以继续使用英文，便于保持 C 工程风格和嵌入式代码可读性。
- 如果引用原始工程中的英文枚举、宏、函数名，应保持原名，不做翻译。

## 编码规则

- 所有新增文件统一使用 UTF-8 编码。
- 不再使用 GBK、ANSI 或混合编码创建新文件。
- 从旧工程迁移内容时，如果旧文件不是 UTF-8，应在 v5.0 新项目中重新整理为 UTF-8。

## 注释规则

- 注释解释“为什么这样做”和“硬件/业务含义”，不写空洞重复说明。
- 电平含义必须写清楚，例如“低电平有效”“高电平表示异常”。
- 关键状态机必须写明状态转换条件。
- 与旧工程行为不同的地方，必须在注释或文档中说明。

## 文档规则

- 每个阶段的实现都要更新对应文档。
- 新旧行为差异必须记录。
- 硬件验证点必须记录。
- 不确定的地方必须标注“待确认”，不能写成结论。
