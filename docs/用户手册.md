# 三通道切换箱控制系统用户手册

版本日期：2026-06-17

## 1. 产品简介

三通道切换箱控制系统用于控制三路高压切换通道。

系统具备：

- 三路外部使能输入。
- 三路磁保持继电器控制。
- 每路双继电器状态反馈。
- 每路接触器状态反馈。
- OLED 状态显示。
- 外部 PLC 控制电源监控。
- ALARM 报警输出。
- 蜂鸣器报警。
- 三路温度监控。
- 风扇调速控制。

系统设计目标是：在保证三路互锁的前提下，根据外部使能信号控制对应通道，并实时监控反馈、温度和电源状态。

## 2. 工作状态说明

### 上电启动

系统上电后：

1. OLED 居中显示缩小后的 LOGO，持续 2 秒。
2. LOGO 显示期间，系统进行基础初始化、输入采样预热和复位原因记录，但不会输出继电器动作，也不会给出最终业务报警结论。
3. 系统进入自检流程，自检界面显示小 LOGO、3 秒进度条和当前自检项目。
4. 系统读取外部使能输入和内部反馈状态。
5. 系统根据当前状态决定进入待机、接管运行、执行安全关断或报警。

说明：

- LOGO、自检和继电器动作等待均不会阻塞主循环。
- 系统发生 A~O 业务异常时，会进入报警显示和报警输出逻辑，但这不等同于软件内部错误。
- 业务异常解除后，系统按当前稳定输入和当前反馈重新判断，不执行异常期间的旧请求。

### 待机状态

当三路外部使能均未触发，且三路反馈均为关闭状态时，系统进入待机状态。

待机状态下：

- 三路通道均关闭。
- ALARM 不报警。
- 蜂鸣器不报警。
- OLED 显示当前通道状态、温度和风扇转速。
- OLED 使用英文或缩写显示，不使用中文。
- OLED 需要同时显示三路温度。
- OLED 使用脏区刷新机制，只刷新变化区域，减少屏幕闪动。

### 通道运行状态

当外部只使能一路通道时，系统会控制对应通道打开。

例如：

- `K1_EN` 有效时，系统打开通道 1。
- `K2_EN` 有效时，系统打开通道 2。
- `K3_EN` 有效时，系统打开通道 3。

系统禁止多路通道同时打开。

三路使能信号全部无效时，系统目标状态为全部关闭。

系统不支持通道间直接自动切换。

如果需要从通道 1 切换到通道 2，外部设备必须先取消通道 1 使能，等待系统关闭通道 1 并确认反馈关闭后，再使能通道 2。

通道 2、通道 3 切换到其他通道时同理。

如果当前通道仍处于打开状态，而外部设备直接请求另一路通道，系统会禁止继电器动作，并进入故障保护。

运行状态下：

- OLED 显示当前运行通道。
- OLED 显示三路通道状态。
- OLED 显示三路温度。
- OLED 显示风扇 PWM 或转速信息。
- 三路通道状态分为三个格，分别居中显示 `CH1`、`CH2`、`CH3`。
- 实际反馈确认打开的通道使用反显形式显示。
- 实际反馈确认关闭的通道使用正常字体显示。
- 继电器动作中和反馈等待中的短暂状态不单独显示。

### 复位后接管状态

如果系统因干扰或看门狗复位，而外部使能信号仍然保持有效，系统重启后会先识别当前外部请求和实际反馈状态。

如果外部请求和反馈一致，系统会接管当前运行状态，不会重新输出继电器动作脉冲，也不会主动关断通道。

说明：

- 看门狗用于处理软件卡死或主循环失控。
- A~O 业务异常本身不会直接导致看门狗复位。
- 出现业务异常时，系统优先通过 ALARM、蜂鸣器、动作禁止或输入冻结处理。
- 看门狗复位后，系统按上电/复位接管策略重新识别现场状态。
- 如果 OLED 或诊断信息显示 `WDT RST`，表示上一次复位可能由看门狗触发。
- `WDT RST` 只表示复位来源，不等同于 A~O 故障，不会单独触发 ALARM 或蜂鸣器。

### 安全关断复位状态

由于磁保持继电器具有掉电保持特性，如果外部设备掉电后关闭所有使能，但继电器仍保持掉电前的闭合状态，系统重新上电后会识别到：

- 外部没有使能请求。
- 某一路继电器仍处于打开反馈。

此时系统会执行该通道的安全关断动作，使继电器恢复到关闭状态。

如果关断成功，系统进入待机。

如果关断失败，系统触发自检异常。

### PLC 控制源异常保持状态

本系统由 UPS 供电，外部 `K1_EN`、`K2_EN`、`K3_EN` 使能信号通常来自 PLC。

`DC_CTRL` 用于检测 PLC 主供电是否正常。

当 PLC 主供电异常时，三个使能信号可能会同时丢失。为了防止磁保持继电器因为 PLC 掉电而被误关断，系统会执行以下策略：

1. 一旦 `DC_CTRL` 疑似异常，立即冻结 `K1_EN`、`K2_EN`、`K3_EN` 控制事件。
2. `DC_CTRL` 按 50ms 间隔连续 3 次确认异常后，触发 O 类严重故障。
3. PLC 电源异常期间，系统不接受三路使能信号控制。
4. 当前继电器状态保持不变。
5. PLC 电源恢复后，系统重新同步外部请求和内部反馈状态。

## 3. IO 点说明

### 外部使能输入

| 信号 | MCU 引脚 | 作用 | 有效状态 |
|---|---|---|---|
| K1_EN | PB9 | 通道 1 使能输入 | 低电平有效 |
| K2_EN | PB8 | 通道 2 使能输入 | 低电平有效 |
| K3_EN | PA15 | 通道 3 使能输入 | 低电平有效 |

说明：

- 下降沿表示通道开启请求。
- 上升沿表示通道关闭请求。
- 系统会对 K_EN 输入进行 150ms 间隔连续 3 次确认，避免误触发。
- K_EN 输入只有在 PLC 控制电源确认正常时才有效。
- PLC 控制电源未确认正常时，系统不会执行新的开启或关闭请求。
- 任意时刻只允许一路 K_EN 有效。
- 切换通道时必须先让三路 K_EN 全部无效，确认当前通道关闭后，再使能目标通道。
- 继电器正在动作或等待反馈确认时，系统不会抢先执行新的开启请求。

### 继电器驱动输出

| 信号 | MCU 引脚 | 作用 | 有效状态 |
|---|---|---|---|
| K1_1_ON | PC0 | 通道 1 第一路继电器开启 | 低电平有效 |
| K1_1_OFF | PC1 | 通道 1 第一路继电器关闭 | 低电平有效 |
| K1_2_ON | PA12 | 通道 1 第二路继电器开启 | 低电平有效 |
| K1_2_OFF | PA3 | 通道 1 第二路继电器关闭 | 低电平有效 |
| K2_1_ON | PC2 | 通道 2 第一路继电器开启 | 低电平有效 |
| K2_1_OFF | PC3 | 通道 2 第一路继电器关闭 | 低电平有效 |
| K2_2_ON | PA4 | 通道 2 第二路继电器开启 | 低电平有效 |
| K2_2_OFF | PA5 | 通道 2 第二路继电器关闭 | 低电平有效 |
| K3_1_ON | PC7 | 通道 3 第一路继电器开启 | 低电平有效 |
| K3_1_OFF | PC6 | 通道 3 第一路继电器关闭 | 低电平有效 |
| K3_2_ON | PD2 | 通道 3 第二路继电器开启 | 低电平有效 |
| K3_2_OFF | PA7 | 通道 3 第二路继电器关闭 | 低电平有效 |

说明：

- 继电器驱动输出为 500ms 低电平脉冲。
- 脉冲结束后系统等待 500ms，再检查状态反馈。
- 同一通道的两路继电器会同时动作。
- 继电器动作期间，系统不会执行其他通道的新动作。

### 继电器状态反馈

| 信号 | MCU 引脚 | 作用 | 有效状态 |
|---|---|---|---|
| K1_1_STA | PC4 | 通道 1 第一路继电器反馈 | 高电平有效 |
| K1_2_STA | PB1 | 通道 1 第二路继电器反馈 | 高电平有效 |
| K2_1_STA | PC5 | 通道 2 第一路继电器反馈 | 高电平有效 |
| K2_2_STA | PB10 | 通道 2 第二路继电器反馈 | 高电平有效 |
| K3_1_STA | PB0 | 通道 3 第一路继电器反馈 | 高电平有效 |
| K3_2_STA | PB11 | 通道 3 第二路继电器反馈 | 高电平有效 |

说明：

- 通道打开时，目标通道两路继电器反馈必须为高电平。
- 通道打开时，非目标通道继电器反馈必须为低电平。
- 通道关闭时，所有继电器反馈必须为低电平。

### 接触器状态反馈

| 信号 | MCU 引脚 | 作用 | 有效状态 |
|---|---|---|---|
| SW1_STA | PA8 | 通道 1 接触器反馈 | 高电平有效 |
| SW2_STA | PC9 | 通道 2 接触器反馈 | 高电平有效 |
| SW3_STA | PC8 | 通道 3 接触器反馈 | 高电平有效 |

说明：

- 通道打开时，目标通道接触器反馈必须为高电平。
- 通道打开时，非目标通道接触器反馈必须为低电平。
- 通道关闭时，所有接触器反馈必须为低电平。
- 任意一个继电器反馈或接触器反馈不符合预期，系统会产生 B~J 对应异常。
- B~J 反馈异常期间，系统不接受新的通道动作指令。
- B~J 反馈异常可在反馈恢复正常一致后自动解除。
- B~J 异常解除后，系统不会执行异常期间曾经出现过的旧请求。
- B~J 异常解除后，系统会根据当前稳定使能信号和当前实际反馈重新判断状态。
- 如果当前使能信号与实际反馈一致，系统只接管当前状态，不重复输出继电器动作。
- 如果当前使能信号请求另一路，但实际反馈显示已有通道打开，系统会按非法直接切换进入保护。

### 电源、报警和蜂鸣器

| 信号 | MCU 引脚 | 作用 | 有效状态 |
|---|---|---|---|
| DC_CTRL | PB5 | 外部 PLC 控制电源监控 | 高电平表示异常 |
| ALARM | PB4 | 异常报警输出 | 低电平有效 |
| BEEP | PB3 | 蜂鸣器输出 | 低电平有效 |

说明：

- `DC_CTRL=0` 表示 PLC 控制电源正常，三路 K_EN 使能信号有效。
- `DC_CTRL=1` 表示 PLC 控制电源丢失或异常，三路 K_EN 使能信号无效。
- 系统会先判定 DC_CTRL，再处理 K_EN。
- 一旦 DC_CTRL 疑似异常，系统会立即冻结 K_EN 控制，避免 PLC 掉电造成误关断。
- PLC 控制电源异常期间，系统保持当前继电器状态，不会因为 K_EN 丢失而关断继电器。
- 任意异常发生时，ALARM 输出低电平。

### 温度和风扇

| 信号 | MCU 引脚 | 作用 |
|---|---|---|
| NTC_1 | PA0 | 通道 1 温度采集 |
| NTC_2 | PA1 | 通道 2 温度采集 |
| NTC_3 | PA2 | 通道 3 温度采集 |
| FAN_PWM | PA6 | 风扇 PWM 控制 |
| FAN_SEN | PC12 | 风扇转速反馈 |

温度规则：

- 风扇根据三路温度中的最高值控制。
- 最高温度大于等于 35℃：风扇 PWM 95%。
- 最高温度下降到 33℃ 以下：风扇 PWM 50%。
- 温度大于等于 60℃ 且连续确认后：产生对应温度异常。
- 温度降到 58℃ 以下且连续确认后：解除对应温度异常。
- 温度异常只作为报警提示，不主动关断通道。
- 温度异常期间，通道开启和关闭仍按外部请求正常执行。
- NTC 断线、短路或换算失败时，归入对应 K/L/M 温度异常，并在显示中提示温度传感器异常。
- 风扇测速异常第一版只作为诊断信息或显示提示，不新增 A~O 异常。

## 4. 异常说明

系统支持 A~O 共 15 类异常。

| 异常 | 含义 | 故障等级 |
|---|---|---|
| A | 多路使能冲突 | 严重故障 |
| B | K1_1_STA 反馈异常 | 紧急故障 |
| C | K2_1_STA 反馈异常 | 紧急故障 |
| D | K3_1_STA 反馈异常 | 紧急故障 |
| E | K1_2_STA 反馈异常 | 紧急故障 |
| F | K2_2_STA 反馈异常 | 紧急故障 |
| G | K3_2_STA 反馈异常 | 紧急故障 |
| H | SW1_STA 反馈异常 | 紧急故障 |
| I | SW2_STA 反馈异常 | 紧急故障 |
| J | SW3_STA 反馈异常 | 紧急故障 |
| K | NTC_1 温度异常 | 一般故障 |
| L | NTC_2 温度异常 | 一般故障 |
| M | NTC_3 温度异常 | 一般故障 |
| N | 自检异常或状态冲突保护 | 严重故障 |
| O | 外部 PLC 控制电源异常 | 严重故障 |

### 报警表现

| 故障等级 | 异常类型 | ALARM 输出 | 蜂鸣器 |
|---|---|---|---|
| 严重故障 | A、N、O | 异常存在期间持续低电平 | 长鸣 |
| 紧急故障 | B~J | 异常存在期间持续低电平 | 100ms 响、100ms 停循环 |
| 一般故障 | K~M | 异常存在期间持续低电平 | 500ms 响、500ms 停循环 |

说明：

- ALARM 不区分异常等级。
- 只要存在任意 A~O 异常，ALARM 就持续输出低电平。
- 所有异常解除后，ALARM 立即恢复高电平。
- 多个异常可以同时存在，系统会保留所有未解除的异常。
- 蜂鸣器按当前最高异常等级输出不同报警时序。
- 最高异常等级变化时，蜂鸣器立即切换到新的报警时序，并从响声开始。
- 所有异常解除后，蜂鸣器立即停止。
- 影响动作安全的异常存在时，系统不会接受新的通道动作指令。

## 5. 用户级故障排查指南

### A 类异常：多路使能冲突

可能原因：

- 外部控制设备同时给了两路或三路使能。
- 接线错误导致多路 `K_EN` 同时为低电平。
- 外部控制信号受到干扰。

处理方法：

1. 检查外部控制逻辑，确保同一时间只允许一路通道使能。
2. 测量 `K1_EN`、`K2_EN`、`K3_EN` 电平。
3. 确认只有一路为低电平，或三路均为高电平。
4. A 类异常解除后，系统不会执行异常期间曾经出现过的旧请求。
5. 如需重新开启通道，应先保持三路 `K_EN` 全部为高电平，确认报警解除后，再重新给出单路使能。

### B~G 类异常：继电器反馈异常

可能原因：

- 磁保持继电器未动作到位。
- 继电器反馈线路接线异常。
- 驱动输出未正确输出 500ms 脉冲。
- 继电器本体故障。

处理方法：

1. 查看 OLED 显示的具体异常编号。
2. 检查对应继电器反馈信号。
3. 检查对应 ON/OFF 驱动输出。
4. 检查继电器供电和外部接线。
5. 排除故障后，使三个通道反馈恢复到正常一致状态。
6. 反馈异常解除前，系统不会接受新的通道动作指令。
7. 异常解除后，需要重新稳定给出通道使能，系统才会响应。

### H~J 类异常：接触器反馈异常

可能原因：

- 接触器未吸合或未断开。
- 接触器反馈线路异常。
- 外部主回路未达到预期状态。

处理方法：

1. 检查对应接触器实际状态。
2. 测量 `SW1_STA`、`SW2_STA`、`SW3_STA`。
3. 检查接触器控制回路。
4. 排除故障后，使三个通道反馈恢复到正常一致状态。
5. 反馈异常解除前，系统不会接受新的通道动作指令。
6. 异常解除后，需要重新稳定给出通道使能，系统才会响应。

### K~M 类异常：温度异常

可能原因：

- 对应通道温度过高。
- 风扇故障或风道堵塞。
- NTC 接线异常。
- NTC 传感器损坏。

处理方法：

1. 检查 OLED 显示温度。
2. 检查风扇是否转动。
3. 检查散热风道。
4. 等待温度降至 58℃ 以下。
5. 如温度显示明显异常，检查 NTC 线路。
6. 如果显示温度传感器异常，检查对应 NTC 是否断线、短路或接触不良。

说明：

- K~M 属于低风险提示型异常。
- K~M 异常期间，系统不会主动关闭已经打开的通道。
- K~M 异常期间，系统仍会按外部请求执行通道开启和关闭。

### N 类异常：自检异常或状态冲突保护

可能原因：

- 上电时内部反馈状态异常。
- 磁保持继电器残留状态无法安全关断。
- 多路反馈同时打开。
- 温度或反馈不满足自检条件。
- 当前通道尚未关闭，外部设备直接请求另一路通道。

处理方法：

1. 关闭外部所有使能信号。
2. 确认 `K1_EN`、`K2_EN`、`K3_EN` 全部为高电平。
3. 等待系统关闭当前残留通道。
4. 确认所有继电器反馈和接触器反馈均为关闭状态。
5. 确认 N 类异常解除。
6. 需要重新开启通道时，只给目标通道一路使能。
7. 如果仍报警，检查继电器反馈和接触器反馈。

非法直接切换恢复方法：

1. 取消所有通道使能，使 `K1_EN`、`K2_EN`、`K3_EN` 全部为高电平。
2. 等待当前通道完成关闭动作。
3. 确认 OLED 显示当前无通道打开。
4. 确认 N 类异常解除。
5. 再使能目标通道。

### O 类异常：外部 PLC 控制电源异常

可能原因：

- 外部 PLC 主供电断开。
- PLC 控制电源电压异常。
- DC_CTRL 检测线路异常。
- 接线松动。

处理方法：

1. 检查外部 PLC 主供电是否正常。
2. 测量 `DC_CTRL` 电平。
3. PLC 控制电源正常时 `DC_CTRL` 应为低电平。
4. 恢复 PLC 控制电源后，系统会重新同步 K_EN 和继电器反馈状态。

注意：

- O 类异常期间，三路 K_EN 信号不会控制继电器。
- O 类异常期间，系统不接受新的继电器开关动作。
- 如果 PLC 掉电导致 K_EN 全部变为高电平，系统不会因此主动关闭正在工作的磁保持继电器。

## 6. 使用注意事项

### 维护按键和日志

设备当前使用两颗本地维护按键：

| 按键 | 引脚 | 用户功能 |
|---|---|---|
| KEY1 短按 | PC13 | 通过调试串口调出日志，或回到上一页 |
| KEY2 短按 | PC14 | 翻到下一页日志 |
| KEY1 长按 | PC13 | 退出日志分页输出状态 |
| KEY2 长按 | PC14 | 清空日志记录 |

PC15 当前不作为按键使用，已作为 10Hz 运行心跳输出。外部 PLC 可通过检测该 10Hz 方波判断控制器主循环是否仍在正常运行；如果 MCU 复位、程序卡死或主循环停止，该心跳会中断或不再保持 10Hz。

日志说明：

- 日志主要用于调试串口维护。
- 日志最多保留 2000 条，超过后新记录会覆盖最旧记录。
- 每页最多显示 50 条日志。
- 每显示一页后暂停，等待按键翻页。
- 日志时间从每次上电启动开始计时，单位为秒。
- 日志只在事件发生时记录，不会每秒固定生成一条记录。
- 日志会记录上电、复位原因、实际发生动作的通道、异常产生、异常解除和日志清空等事件。
- 未动作通道的维持状态不会写入日志。
- 温度正常变化、风扇 PWM 变化和风扇转速变化不会写入日志；只有触发 A~O 异常时才记录。
- 如果系统发生看门狗复位，下次启动后会记录看门狗复位事件。
- 如果系统能够在复位前保存软件原因，日志会显示对应原因；如果不能确认，则显示未知原因。
- 清空日志后，系统会保留一条新的 `LOG_CLEARED` 记录，用于说明历史日志是被维护操作清空。

注意：

- 两颗维护按键不会静音蜂鸣器。
- 两颗维护按键不会强制清除当前真实存在的故障。
- KEY1、KEY2 长按必须持续满 3 秒才触发；如果 3 秒内松开，不触发长按动作，也不会转成短按动作。
- 清空日志只清除历史记录，不改变当前 ALARM、蜂鸣器、继电器和通道状态。

- 禁止同时使能多路通道。
- 出现严重故障长鸣时，应立即检查外部使能、电源和自检状态。
- 出现紧急故障 100ms 响、100ms 停循环蜂鸣时，应优先检查继电器和接触器反馈。
- 出现一般故障 500ms 响、500ms 停循环蜂鸣时，应检查温度和散热系统。
- 不建议在故障未排除前反复上电重启。
- 外部设备掉电恢复后，如果系统执行安全关断动作，应确认通道状态恢复到预期。
